# Output Schema

Per-source outputs are written under `results/`:

- `spectra/<source_name>.csv`
- `plots/<source_name>_sed.png`
- `qa/<source_name>/<source_name>_qa_summary.png`
- `qa/<source_name>/measurements/<measurement_id>_qa.png` for full QA
- `provenance/<source_name>_provenance.json`
- `provenance/<source_name>_measurement_index.json`
- `provenance/<source_name>_output_manifest.json`

When full QA is requested, the output manifest includes a `full_qa` section
with the expected measurement IDs, PNG paths, file sizes, and SHA256 checksums.
Compact source outputs can be current without full QA; full-QA freshness is
validated separately so stale or missing diagnostic PNGs can be regenerated
without redownloading science cutouts.

Important CSV column groups:

- identity: source, product, cutout, detector, measurement IDs
- spectral metadata: wavelength, bandwidth, observation/time fields
- point-source baseline: `point_flux_uJy`, `point_flux_err_uJy`, `point_snr`
- selected result: `selected_flux_uJy`, `selected_flux_err_uJy`, `science_mode`
- state: `detection_status`, `science_recommended`, flags
- QA: mask fractions, per-region flag fractions, PSF-weighted bad-pixel
  fractions, invalid-variance fractions, fit condition number, fit quality, and
  background/deblend status
- provenance: calibration IDs, config hash, schema and code versions

The output manifest is the freshness authority. File existence alone is not
treated as current output.

Unit conventions:

- wavelength and bandwidth columns ending in `_um` are microns;
- flux and flux-uncertainty columns ending in `_uJy` are microJy;
- MJD columns are copied from product metadata when available;
- `input_image_extension` is `IMAGE` and `zodi_used` is `false` in the default
  science path;
- redshift/rest-frame and Galactic-extinction-corrected quantities are not
  generated by default.

When comparing or combining outputs, require compatible `config_hash`,
`spectral_wcs_calibration_id`, `solid_angle_calibration_id`,
`output_schema_version`, and `photometry_code_version` values. Mixing old
official FITS products with newer local CSV products without these checks can
produce scientifically inconsistent spectra.
